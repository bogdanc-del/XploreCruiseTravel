## 4. User Journey Analysis

### 4.1 Primary Path — Guided Recommendation (Optimized)

```
Homepage → Click "Quick Recommendation"
        → Step 1: Travel party (10 sec)
        → Step 2: Experience (10 sec)
        → Step 3: Priority (10 sec)
        → Step 4: Timing (10 sec)
        → (Optional) Step 5: Budget + Destination
        → Results: 3-5 recommended cruises
        → Click "Solicita oferta"
        → Fill lead form (30 sec)
        → Submission → Consultant follow-up (< 24h)
```

**Estimated time:** 60-90 seconds from landing to lead
**Data collected:** Travel party, experience level, priority, timing, (budget, destination)
**Consultant benefit:** Full context before first call

### 4.2 Secondary Path — Self-Service Browse

```
Homepage → Click "Explore all cruises" or nav link
        → /cruises listing with 8,400+ options
        → Apply filters (destination, type, price, duration)
        → Search by keyword
        → Click cruise card → Detail page
        → Click "Solicita oferta"
        → Fill lead form
        → Submission → Consultant follow-up
```

**Estimated time:** 2-5 minutes (browsing)
**Data collected:** Cruise interest, contact info
**Best for:** Users who know what they want

### 4.3 Chat-Assisted Path

```
Any page → Click chat widget (bottom-right)
        → Ask "Daniela" questions
        → Receive recommendations
        → Navigate to recommended cruise
        → Submit lead form
```

**Estimated time:** Variable (conversation length)
**Best for:** Users with specific questions

### 4.4 QR Review Path (Post-Purchase)

```
Customer receives QR code → Scan → /review page
        → Submit star rating + text review
        → Admin moderates (approve/reject)
        → Approved reviews appear on site
        → New visitors see social proof → Trust → Convert
```

**Purpose:** Build social proof flywheel

## 7. Risk Analysis

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Low organic traffic initially | High | Medium | SEO optimization, 8,400+ indexed pages, sitemap |
| AI chat costs escalate | Medium | Low | Claude API usage monitoring, rate limiting |
| Cruise data staleness | Low | Medium | Automated daily sync via GitHub Actions (04:00 UTC), freshness labels |
| BNR rate unavailable | Low | Low | Fallback chain: 10-day feed → hardcoded rate (4.97) |
| Competitor launches similar site | Low | Medium | Differentiation via consulting + AI + guided flow |
| GDPR complaint | Low | High | Consent checkboxes, privacy policy, data handling policies |
| Supabase downtime | Low | Low | Demo data fallback for all features |
