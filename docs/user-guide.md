## 1. Getting Started

Welcome to XploreCruiseTravel, your trusted cruise travel partner. Our website allows you to browse cruise offers, compare destinations, and request personalized offers from our expert consultants.

### Accessing the Website
- Open your web browser (Chrome, Firefox, Safari, or Edge).
- Navigate to the XploreCruiseTravel website URL.
- The homepage loads with featured cruises, popular destinations, and a search module.

### What You Can Do
- **Browse** cruise offers from top cruise lines (MSC, Costa, NCL, Viking, Royal Caribbean, Silversea, and more).
- **Search and filter** cruises by destination, duration, cruise line, and month.
- **View detailed information** about each cruise, including itineraries, cabin prices, and what is included.
- **Request a personalized offer** through our 3-step booking form.
- **Contact us** via phone, email, WhatsApp, or the contact form.
- **Switch language** between English and Romanian at any time.

## 8. Understanding Prices (EUR + RON)

### Dual Currency Display
All prices on the site are shown in two currencies:
- **EUR** (Euro) — The primary pricing currency used by cruise lines.
- **RON** (Romanian Leu) — The approximate equivalent for Romanian customers.

### Conversion Rate
The RON equivalent is calculated using a fixed conversion rate of **1 EUR = 4.97 RON**. This is an approximate rate for reference purposes.

### Example
If a cruise is priced at **899 EUR**:
- RON equivalent: 899 x 4.97 = approximately **4,468 RON**

### Important Notes
- Prices shown are **starting prices** ("from" pricing).
- Final prices depend on cabin type, departure date, and availability.
- The RON amount is approximate — the actual exchange rate at the time of booking may differ.
- Your consultant will provide the exact final price in your offer.

## 11. Admin Overview

The XploreCruiseTravel admin interface allows you to manage cruise offerings displayed on the public website. As an administrator, you can:
- View all cruise listings.
- Add new cruises with bilingual content.
- Edit existing cruise details (prices, itineraries, descriptions).
- Delete cruises that are no longer available.
- Manage the data that powers the entire website.

### How Data Works
The website is powered by a single JSON data file: `data/cruises.json`. This file contains:
- **Cruises** — Individual cruise offerings with all details.
- **Destinations** — Destination categories with cruise counts.
- **Cruise Lines** — Available cruise line brands.

All changes to cruise data are reflected on the public site after the data file is updated.

## 17. Managing Bilingual Content

### How Bilingual Content Works
The website supports English and Romanian through a translation system:

1. **Static text** (navigation, buttons, labels) is managed in `js/i18n.js` with translation keys.
2. **Dynamic content** (cruise data) uses paired fields in `cruises.json` — for every English field, there is a Romanian equivalent with the `Ro` suffix.

### Translation Pairs in Cruise Data
| English Field | Romanian Field |
|---|---|
| `title` | `titleRo` |
| `destination` | `destinationRo` |
| `departurePort` | `departurePortRo` |
| `ports` | `portsRo` |
| `included` | `includedRo` |
| `excluded` | `excludedRo` |
| `advisorNote` | `advisorNoteRo` |

### Translation Best Practices
- Always fill in BOTH language fields. If a Romanian translation is missing, the system falls back to English.
- Use natural Romanian phrasing, not word-for-word translations.
- Keep translations consistent — use the same terms for recurring concepts (e.g., "Pensiune completă" for "Full board" throughout).
- Romanian diacritics (ă, â, î, ș, ț) should be used correctly.

### Adding New Static Translations
If you need to add new static text to the site:
1. Open `js/i18n.js`.
2. Add a new key to both the `en` and `ro` objects inside the `translations` constant.
3. Use the `data-i18n="your_key"` attribute on the HTML element.
4. The text automatically translates when the user switches language.

## 18. Data Scraper (Croaziere.Net)

### Overview
The project includes a Python scraper (`scripts/scraper.py`) designed to fetch cruise data from Croaziere.Net, the partner platform.

### How It Works
1. The scraper reads the Croaziere.Net website for available cruises.
2. Data is extracted and formatted to match the `cruises.json` structure.
3. A merge strategy is applied: scraped data is combined with manual entries, preserving any manually added cruises.
4. Backups of the existing data are created before any merge.

### Running the Scraper
```bash
cd scripts
python scraper.py
```

### Important Notes
- The scraper includes a **2-second rate limit** between requests to be respectful to the source server.
- Backups are stored in `data/backups/` with timestamps.
- Manual cruises (those not from the scraper) are preserved during merges.
- Review scraped data before publishing to the live site.
- The scraper is a supplementary tool — manual data entry remains the primary method.

## 19. EmailJS Configuration

### Overview
The website uses EmailJS to send form submissions (booking requests and contact messages) to the business email without a backend server.

### Configuration Details
- **Service ID:** `service_xplore`
- **Booking Template:** `template_booking`
- **Contact Template:** `template_contact`
- **Public Key:** Must be configured (currently placeholder `YOUR_PUBLIC_KEY`)

### Setup Steps
1. Create an account at [emailjs.com](https://www.emailjs.com/).
2. Add your email service (Gmail: xplorecruisetravel@gmail.com).
3. Create two email templates:
   - `template_booking` — For booking requests. Include variables: `{{name}}`, `{{email}}`, `{{phone}}`, `{{cruise}}`, `{{cabin}}`, `{{guests}}`, `{{requests}}`, `{{price}}`.
   - `template_contact` — For contact form submissions. Include variables: `{{name}}`, `{{email}}`, `{{phone}}`, `{{cruise}}`, `{{message}}`.
4. Copy your Public Key from the EmailJS dashboard.
5. Open `js/app.js` and replace `'YOUR_PUBLIC_KEY'` with your actual key:
   ```javascript
   const EMAILJS_PUBLIC_KEY = 'your_actual_public_key_here';
   ```
6. Test both forms to verify emails are received.

### Fallback System
If EmailJS fails (network issue, misconfigured key, etc.):
1. The form data is saved to `localStorage` as a backup.
2. A `mailto:` link is generated as a fallback, allowing the user to send the data via their email client.
3. This triple-send approach (EmailJS + localStorage + mailto) ensures no booking request is lost.

## 20. Deployment & Maintenance

### Hosting
The site is configured for deployment on **Vercel** (see `vercel.json`).

### Deployment Configuration
- **Build:** Static files served by `@vercel/static` — no build step needed.
- **Security Headers:** Configured automatically:
  - X-Content-Type-Options: nosniff
  - X-Frame-Options: DENY
  - X-XSS-Protection: 1; mode=block
  - Referrer-Policy: origin-when-cross-origin
- **Caching:**
  - CSS/JS files: 1 year, immutable (change filenames to bust cache).
  - Data files (cruises.json): 1 hour cache.

### Updating the Site
1. Edit the files locally (HTML, CSS, JS, or data).
2. Test locally by opening `index.html` in a browser.
3. Commit changes to the Git repository.

## 8. Înțelegerea Prețurilor (EUR + RON)

### Afișare în Două Monede
Toate prețurile de pe site sunt afișate în două monede:
- **EUR** (Euro) — Moneda principală utilizată de liniile de croazieră.
- **RON** (Leu Românesc) — Echivalentul aproximativ pentru clienții români.

### Rata de Conversie
Echivalentul RON este calculat folosind o rată fixă de **1 EUR = 4,97 RON**. Aceasta este o rată aproximativă de referință.

### Exemplu
Dacă o croazieră costă **899 EUR**: echivalentul RON = 899 x 4,97 = aproximativ **4.468 RON**.

### Note Importante
- Prețurile afișate sunt **prețuri de pornire** (preț „de la").
- Prețul final depinde de tipul cabinei, data de plecare și disponibilitate.
- Suma RON este aproximativă — rata de schimb reală la momentul rezervării poate diferi.
- Consultantul dumneavoastră vă va furniza prețul final exact în ofertă.
